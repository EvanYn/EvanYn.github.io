从 MATLAB 创建外部可修改参数的独立应用程序
2019-10-21|笔记
字数总计: 1.4k|阅读时长: 4 分钟

问题描述

通常我们创建的 .m 脚本程序是在 MATLAB 环境中运行的，如果要在没有安装 MATLAB 的电脑中运行这个程序，并且要满足两个要求：(1) 要像在 .m 文件中一样，程序的参数可修改；(2) matlab 源代码要保密。这时该怎么办？

解决方案

先用 MATLAB 提供的 MATLAB Compiler 应用将 MATLAB 程序编译为独立、可共享的应用程序，然后使用 MATLAB Runtime 部署这些应用程序。具体说来有以下几步：

配置过程如下：

  1. 在 Add main file 区域添加主程序 test.m.
  2. Application Compiler 会自动在 Files required for your application to run 区域中加载主程序所需的附加文件 parameter.txt.
  3. 选择 Runtime included in package, 这个选项在编译时会将运行应用程序所需的 MATLAB Runtime（关于 MATLAB Runtime 下小节会详细介绍） 和 MATLAB 脚本文件打包在一起。将编译后的应用程序部署到另一台没有安装 MATALB 的电脑、或没有安装相对应版本 MATLAB Runtime 的电脑时，就不用再次安装对应版本的 MATLAB Runtime了。否则，若选择另一个 Runtime downloaded from web 选项，在另一台电脑上运行编译后的应用程序时，还需要下载大约 1.93 G 的 MATLAB Runtime.
  4. 点击 Package 开始编译。

部署编译所得的应用程序

在另一没有安装 MATLAB 的电脑上运行编译所得的应用程序需要借助 MATLAB Runtime. MATLAB Runtime 是共享库和代码的集合，这些库和代码可在未安装 MATLAB 的系统上执行已编译和打包的 MATLAB 应用程序；如果终端用户的电脑上没有安装 MATLAB, 但想要运行已编译和打包的 MATLAB 应用程序，必须在其计算机上安装MATLAB Runtime 或知道网络安装 MATLAB Runtime 的位置。MATLAB 和 MATLAB Runtime 的区别如下：

  • 在 MATLAB Runtime 中，MATLAB 文件是加密的并且是不更改的，这正好实现了我们所需的保密要求。
  • MATLAB 具有桌面图形界面，而 MATLAB Runtime 具有 MATLAB 的所有功能，但没有图形界面。
  • MATLAB Runtime 是与版本有关的。 你必须使用与编译应用程序 MATLAB Compiler 版本相关联的 MATLAB Runtime 版本运行该应用程序。 例如，如果使用 MATLAB Compiler 的6.3版（R2016b）编译了应用程序，则未安装 MATLAB 的用户必须安装 9.1 版的 MATLAB Runtime 。

从外部修改程序参数

按照以上过程，应用程序中的参数还是无法修改的，比如魔方矩阵维数还是固定为 3, 这是因为在编译时保存参数的 parameter.txt 文本文件和 MATLAB 程序打包在一起了，见图 3 中 Files required for your application to run 区域。

编译时可先将 parameter.txt 移动到另一位置，这样 Application Compiler 就不会将其自动打包了。然后在部署时，将应用程序与 parameter.txt 置于同一目录，修改 parameter.txt 中的参数，就可进而修改应用程序内的参数了。比如将 parameter.txt 内的 n 设为 4, 运行程序，所得图像显示的就是 4 维魔方矩阵了。
